# The Dream...

![Architecture](architecture.png)

- Let's stop re-inventing the wheel.

- Enterprise software is often just a 90% fit. I want to download an AI-native 90%, so I can use AI to quickly add _my_ 10%.

- My framework should provide logging, database, security, and performance, out of the box.

- I want deployment options - free for low bandwidth, all the way to massive Enterprise scaleout.

- If I write a great 90% "core" application that people build on top of, I should be paid.

- I want to be paid if I add my "secret sauce" on top of an existing product.

- I want to sign on to an ecosystem once, and have all my applications there in one place.

- AI can generate enterprise-level software quickly. Why are we paying so much?

- Open source is fantastic, but tens of thousands of products have essentially the same core. Can't that core stuff be written once, given to all the world to share, and then we can all build on top of it?

- A common situation for Enterprises is that they need to customize their Enterprise software. They spend millions makign changes, but it creates a probelm. They cannot update to a new version of the Enterprise software because they'd need to spend millions to re-do the customizations.

This is bad for the customer, but it's even worse for the vendor because they stuck in the past, having to support multiple obsolete versions of their software, because their customers cannot upgrade.

## The solution

1. Layered application architecture, so applications can build upon each other.

2. Mega-branching, to encourage a hundred flavors of each application.

3. Seamless from development to deployment.

4. Ability to intermix public and private layers.

5. Authentication, logging, database, security, performance, before you even start.

6. Catalog and App Store, payments and commission.

7. AI-Native EVERYTHING.

8. Development and Contribution Gates to ensure compliance with specification, testing, and other requirements.
